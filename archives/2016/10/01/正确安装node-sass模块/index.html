正确安装node-sass模块

Posted on 2016-10-01

安装node-sass模块

在使用webpack打包初始化项目的时候,因为main.js文件中引用了sass文件,build的时候提示缺少sass-loader模块,原始麻溜去install了,当时提示。。大概是sass-loader依赖于node-sass模块,没有理会,重新build的时候,有提示缺少node-sass模块,神烦~麻溜去 npm install node-asss –save-dev 开始还好,提示在github上下载一个win32-x64-48_binding.node文件的时候老是卡住,然后就报错了。安装了不下十遍,各种百度,终于解决了,赶快记下来。

分析

  • 安装node-sass模块的时候,node_modules文件夹下会出现node-sass目录,一旦安装失败,就移除了
  • 在安装的过程中进入node-sass目录,找到package.json文件,打开找到’script’字段下的’install’字段对应的文件为scripts文件下的install.js,打开可以看到,其实要下载的就是github上编译好的二进制包win32-x64-48_binding.node,貌似是被墙了,文件平均2M用的还是request请求,会很慢。。所以会失败。知道他的原理就好。
  • 我们先去node-sass的github上把文件下载下来。保存的本地,然后起一个服务,例如”http://localhost:8080/win32-x64-48_binding.node“, 保证能访问到这个文件就可以
  • 然后找到install.js中的代码
    download(sass.getBinaryUrl, sass.getBinaryPath(), function(err)....
    然后把sass.getBinaryUrl替换为我们的地址’http://localhost:8080/win32-x64-48_binding.node‘, 保存。
    进入到node_modules/node-sass 执行npm install 发现很简单就安装完成了。。。。
  • cover it

    关于本地起服务

  • 我用了express,主要是懒得写多一行代码
  • 新建文件 npm install express,新建server.js,把下载好的win32-x64-48_binding.node文件都放在同一个文件夹里

  • server.js中写入

  • cmd 下node server.js

  • 打开’localhost://8000/win32-x64-48_binding.node’发现下载成功,就可以了。
